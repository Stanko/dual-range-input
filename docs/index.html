Native Dual Range Input

GitHub

The native part is somewhat open for discussion. But the implementation uses two native range inputs and about eighty lines of TypeScript to make them work together. In my book, it is native enough.

How it works

On change, mid-point is calculated, inputs are resized, and their min and max attributes are adjusted.

Here you can see the unstyled example. Even like this, it works reasonably well, but it is somewhat jittery.

When a user clicks between the thumbs, because the inputs are resized to the mid-point, the user will move the thumb that is closer to the point of the click.
